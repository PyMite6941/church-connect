Connect your church in 5 steps

Stand up your own instance and point it at your subdomain.

Add your church

Create a tenant entry — name, accounts, and which functions you offer. This is the only "config" a church needs.

// src/config/tenants.js
"yourchurch": {
  id: "yourchurch",
  name: "Your Church",
  tagline: "Welcome home.",
  features: ["events", "sermons", "prayer", "leadership", "groups", "meals", "giving"],
  accounts: [
    { id: "u-admin", name: "Pastor", email: "pastor@yourchurch.org",
      role: "admin", pin: "1234" }
  ],
}

Make it yours

Match your brand. Every value is optional — set only what you want to change. Rename functions with featureLabels.

theme: {
  primary: "#2b8a3e",     // buttons, links, accents
  accent:  "#e8590c",
  font:    "Georgia, serif",
  radius:  "16px",
},
featureLabels: { meals: "Potlucks" },
logoUrl: "https://yourchurch.org/logo.png",

Deploy the app

Push to any static host (Vercel, Netlify, Cloudflare Pages). One build serves every church.

npm install
npm run build      # -> dist/   (deploy this folder)

Point your subdomain

Add a CNAME record at your DNS provider, then attach the domain in your host. Church Connect reads the subdomain and loads the matching tenant automatically.

TypeNameValue
CNAMEconnectcname.vercel-dns.com
# Result: connect.yourchurch.org  ->  tenant "yourchurch"
# (subdomain label is matched to the tenant id)

Go live & invite your team

Share the URL. Admins sign in and manage everything from the UI — accounts on the Accounts page, content on each function page. Members get their own My Account page.

https://connect.yourchurch.org   ✅ live

Two ways to run it

Host it as a full site, or embed it inside your existing website.

① Standalone subdomain

The full app on connect.yourchurch.org. Best if you don't already have a site.

npm run build        # -> dist/
# deploy dist/ + point the CNAME (steps above)

② Embed widget

Drop it into a page on your current website — no subdomain needed.

npm run build:embed  # -> church-connect.js

<div id="church-connect" data-tenant="yourchurch"></div>
<script src="https://cdn.yoursite/church-connect.js"></script>

Where does the data live?

Out of the box, everything runs on the browser's local storage — zero backend, perfect for trying it out. When you're ready for shared, persistent data, set one env var and point it at your API: 

VITE_API_BASE_URL=https://api.yourchurch.org
# Church Connect then GET/PUTs each collection at /{tenant}/{collection}
